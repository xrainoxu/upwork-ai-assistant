# Upwork Job Analyzer

## 工作方式

你只需要给我指令，我来执行所有操作。

### 1. 更新 Profile

把 Upwork Profile 内容贴给我，说"更新 profile"即可。

---

### 2. 初筛 Jobs

把多个 Job 的摘要列表贴给我，我会：
1. **解析数据** - 提取结构化信息（标题、预算、Client 信息等）
2. **去重** - 排除已分析过的 Job
3. **实时分析** - 基于提取的数据，结合你的 Profile，用 LLM 的能力深度分析
4. **输出结论** - 告诉你每个 Job 是否值得申请，并说明原因

### 3. 详细分析 Job

把单个 Job 的详细信息（含 Client 历史评价）贴给我，我会：
1. **解析数据** - 提取结构化信息
2. **深度分析** - 结合 Client 历史、评价、内容做全面判断
3. **给出建议** - 是否申请 + 报价方案

---

### 示例指令

> "更新 profile，xxx"

> "初筛这些 jobs"

> "详细分析这个 job"

---

## 分层设计

### 脚本层（数据处理）

| 功能 | 说明 |
|------|------|
| 数据解析 | 从原始文本提取：标题、预算、Client 信息、技能标签等 |
| 去重 | 基于 MD5 哈希排除已分析过的 Job |
| 数据存储 | 保存 JSON 供后续使用 |
| 报告生成 | 生成 HTML 报告 |
| 自动打开 | 生成报告后自动在浏览器中打开 |

### LLM 层（实时分析）

| 功能 | 说明 |
|------|------|
| 风险评估 | 结合上下文判断，非固化规则 |
| 技能匹配 | 结合你的 Profile 实时对比 |
| 报价方案 | 根据实际情况给出建议 |
| 细节发现 | 从描述、评价中发现潜在问题 |

## 输出文件位置

### 初筛报告

| 类型 | 位置 |
|------|------|
| HTML 报告 | `reports/screening/screening-{timestamp}.html` |

### 详细分析（文件夹组织）

| 类型 | 位置 |
|------|------|
| 文件夹 | `reports/detailed/job-{id}-{timestamp}/` |
| HTML 报告 | `reports/detailed/job-{id}-{timestamp}/analysis.html` |

### 数据文件

| 类型 | 位置 |
|------|------|
| 解析后数据 | `scripts/jobs/data/jobs-{timestamp}.json` |
| Profile 数据 | `scripts/profile/data/my-profile.json` |
| 已分析任务 | `scripts/jobs/data/analyzed-jobs.json` |

|------|------|

---

## 工作流程

```
你: 贴 Upwork 数据
    ↓
脚本: 解析提取 → JSON
    ↓
LLM: 读取数据 + 读取 Profile → 实时分析 → 输出结论
    ↓
脚本: 生成 HTML 报告 → 自动打开浏览器
    ↓
你: 获得分析结果 + 浏览器查看报告
```

---

## 为什么这样设计

- **脚本擅长**：结构化提取、重复劳动
- **LLM 擅长**：理解上下文、发现细节、归纳总结

让 LLM 做分析，可以发现：
- 描述中的微妙风险信号
- Client 评价中的真实情绪
- 技能匹配度的细微差别
- 预算是否合理的市场判断

而不是被固化的规则限制。
